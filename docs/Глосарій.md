# Глосарій проєкту «Link Collector»

Цей документ містить визначення основних термінів, опис компонентів системи та технічних понять, що використовуються у розробці менеджеру бібліографічних посилань **Link Collector**.

## 1. Предметна область (Domain Concepts)

**Бібліографічне посилання** (*Resource Link*)  
Основна сутність (Entity) системи. Це об'єкт даних, що містить вичерпну інформацію про джерело: унікальний ідентифікатор, назву, автора, рік видання, URL-адресу або опис джерела.

**Тип ресурсу** (*Link Type*)  
Атрибут, що класифікує джерело інформації. Визначає природу матеріалу (наприклад: *Книга*, *Стаття*, *Відео*, *Веб-сайт*, *Документ*). Використовується для фільтрації та правильного форматування при експорті.

**Категорія** (*Category*)  
Користувацька мітка (тег) для логічного групування посилань за тематикою (наприклад: *«Навчання»*, *«Робота»*, *«C# Development»*). Дозволяє структурувати велику кількість записів.

**Стиль цитування** (*Citation Style*)  
Стандартизований набір правил оформлення бібліографічного опису. Визначає порядок полів, пунктуацію та форматування тексту при експорті (наприклад: *APA*, *MLA*, *Harvard*, *Chicago*, *Simple*).

---

## 2. Архітектура та компоненти системи (System Components)

**Головна форма** (*MainForm*)  
Центральний елемент інтерфейсу користувача (UI). Відповідає за відображення таблиці даних (`DataGridView`), панелі інструментів та забезпечує навігацію до функцій створення, редагування, видалення та експорту записів.

**Репозиторій посилань** (*LinkRepository*)  
Статичний клас, що реалізує патерн проєктування *Repository*. Відповідає за збереження даних в оперативній пам'яті (In-memory storage), пошук, додавання, оновлення та видалення записів. Слугує єдиною точкою доступу до даних для всіх форм.

**Сервіс цитування** (*CitationService*)  
Клас бізнес-логіки, відокремлений від інтерфейсу. Відповідає за генерацію форматованого тексту бібліографії на основі обраного стилю та збереження результату у текстовий файл.

**Форма редагування** (*AddEditForm*)  
Універсальне діалогове вікно. Використовується як для створення нових записів (режим `Create`), так і для модифікації існуючих (режим `Update`). Містить логіку валідації введених даних.

**Менеджер категорій** (*CategoryManagerForm*)  
Допоміжна форма для керування списком доступних категорій (додавання нових та видалення застарілих), що забезпечує актуальність випадаючих списків у програмі.

## 3. Технології та інструментарій (Technical Terms)

**Статичне сховище** (*Static Storage / In-memory DB*)  
Підхід до збереження даних, де список об'єктів (`List<T>`) зберігається у статичному полі класу. Дані існують лише під час роботи програми та очищуються при її закритті (імітація бази даних).

**Windows Forms** (*WinForms*)  
Графічна підсистема .NET Framework / .NET Core, використана для побудови інтерфейсу користувача (GUI) даного застосунку.

**DataGridView**  
Складний елемент керування WinForms, призначений для відображення колекцій даних у вигляді таблиці з можливості сортування, виділення рядків та налаштування стовпців.

**Рефакторинг** (*Refactoring*)  
Процес оптимізації внутрішньої структури коду без зміни його зовнішньої поведінки. У контексті проєкту — це винесення логіки з форм (`Forms`) у сервіси (`Services`) та моделі (`Models`) для покращення читабельності та підтримки.

**XML-документація** (*XML Documentation Comments*)  
Спеціальний формат коментарів (починається з `///`), що дозволяє описувати класи та методи. Використовується середовищем розробки (IntelliSense) для підказок та генерації технічної документації.

**Тег** (*Git Tag*)  
Незмінне посилання на конкретний коміт в історії Git. Використовується для маркування релізів (наприклад, `v1.0`), фіксуючи стабільний стан коду в певний момент часу.
