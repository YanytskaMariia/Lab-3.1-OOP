Глосарій проєкту «Link Collector»

Нижче наведено стислий, актуальний та контекстно-залежний глосарій для проєкту Link Collector українською мовою.

Доменні концепції (Domain Concepts)

  • ResourceLink
    • Визначення: Доменна сутність, що представляє один бібліографічний запис (ID, Назва, Автор, Рік, URL/Джерело, Категорія, Тип).
    • Відповідальність: Зберігання даних (DTO / entity).
    • Розташування: Models\ResourceLink.cs.
    • Примітки: Змінний POCO-об'єкт; метод ToString() використовується для простого відображення.
  • LinkType
    • Визначення: Перелік (enum), що класифікує ResourceLink (Книга, Вебресурс, Відео, Стаття).
    • Розташування: Models\Enums.cs.
  • CitationStyle
    • Визначення: Перелік підтримуваних форматів цитування (ДСТУ_8302, Harvard, BibTeX).
    • Розташування: Models\Enums.cs.
  • Category
    • Визначення: Легковага мітка користувача для групування посилань (зберігається як список рядків у репозиторії).
    • Відповідальність: Використовується у випадаючих списках UI та для фільтрації/групування записів.

Інфраструктура та компоненти (Infrastructure & Components)

  • ILinkRepository (інтерфейс)
    • Визначення: Контракт репозиторію (GetAll, Search, Add, Remove, GetCategories, AddCategory, RemoveCategory).
    • Відповідальність: Порт доступу до даних — абстрагує деталі зберігання.
    • Розташування: Services\ILinkRepository.cs.
    • Примітки: Використовувати в сервісах/формах для забезпечення тестопридатності.
  • LinkRepository (реалізація in-memory)
    • Визначення: Конкретний репозиторій у пам'яті, що реалізує ILinkRepository. Заповнює початкові дані в конструкторі.
    • Відповідальність: Зберігання списків ResourceLink та категорій; реалізація валідації при додаванні (Add).
    • Розташування: Services\LinkRepository.cs.
    • Примітки: Наразі зареєстрований як Singleton у DI; методи GetAll()/GetCategories() повертають списки (варто розглянути повернення копій або readonly). Не є потокобезпечним.
  • ICitationService (інтерфейс)
    • Визначення: Контракт для генерації окремих цитат та списків форматованих цитат.
    • Розташування: Services\ICitationService.cs.
  • CitationService (реалізація)
    • Визначення: Форматує екземпляри ResourceLink у вибраний CitationStyle та генерує контент для експорту.
    • Відповідальність: Форматування тексту, генерація ключів BibTeX, форматування списків.
    • Розташування: Services\CitationService.cs.
    • Примітки: Реалізовано як екземпляр класу (без статичних методів).
  • Program (корінь композиції / composition root)
    • Визначення: Точка входу в програму, де будується DI-контейнер та створюється MainForm.
    • Розташування: Program.cs.
    • Примітки: Використовує Microsoft.Extensions.DependencyInjection; поточні реєстрації: ILinkRepository (Singleton), ICitationService (Singleton), MainForm (Transient).

UI та взаємодія (UI & Interactions)

  • MainForm
    • Визначення: Головний контролер UI; відображає DataGridView, панель інструментів та відкриває інші форми.
    • Відповідальність: Координація дій користувача, виклик _repo та _citationService, оновлення інтерфейсу.
    • Розташування: Forms\MainForm.cs.
    • Примітки: Отримує ILinkRepository та ICitationService через конструктор; передає посилання на _repo при створенні дочірніх форм для спільного доступу до стану.
  • AddEditForm
    • Визначення: Діалогове вікно для створення або редагування ResourceLink.
    • Відповідальність: Збір даних від користувача та виклик _repo.Add(...) або зміна існуючого об'єкта.
    • Розташування: Forms\AddEditForm.cs.
    • Примітки: Конструктор підтримує AddEditForm(ILinkRepository repo, ResourceLink link = null). Безпараметричний конструктор (для дизайнера) делегує створення дефолтному LinkRepository — залишити лише для дизайнера.
  • CategoryManagerForm
    • Визначення: Діалог для додавання/видалення категорій.
    • Відповідальність: Виклик _repo.AddCategory / _repo.RemoveCategory та оновлення UI.
    • Розташування: Forms\CategoryManagerForm.cs.
    • Примітки: Приймає ILinkRepository.
  • ExportForm
    • Визначення: Діалог для вибору стилю цитування та збереження форматованого виводу у файл.
    • Відповідальність: Виклик генерації цитат та запис у файл.
    • Розташування: Forms\ExportForm.cs.
    • Примітки: Приймає ILinkRepository, але наразі створює CitationService напряму — рекомендовано впроваджувати ICitationService для узгодженості.

Тестування (Testing & Testability)

  • Тести: Тести репозиторію та сервісу цитування оновлено для роботи з екземплярами, а не статичними методами.
  • Розташування: ..\LinkCollector.Tests\*.
  • Рекомендація: Тестувати проти інтерфейсів ILinkRepository та ICitationService, використовуючи фейки/моки (fakes/mocks) для ізоляції.

Архітектура та принципи (Design & Architecture)

  • Dependency Inversion & DI (Інверсія залежностей):
    • Використання інтерфейсів та ServiceCollection у Program дозволяє інвертувати залежності. MainForm залежить від абстракцій, а не від конкретних реалізацій.
  • Ports & Adapters (Гексагональна архітектура):
    • ILinkRepository та ICitationService діють як порти; LinkRepository/CitationService — як адаптери. Це спрощує заміну механізмів зберігання або експорту.
  • Single Responsibility & Cohesion (Єдиний обов'язок та зв'язність):
    • Чіткий розподіл обов'язків: репозиторій — для зберігання, сервіс цитування — для форматування, форми — для UI.
